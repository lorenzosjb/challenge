REST API

 

POST api/championship/result

Stores the first and second place of a tournament, each user is stored with their respective scores. The user names will be unique, but the same user can win 1 or more championships. Returns the operation status if successfully.

Resource information

  • HTTP Method: POST
  • Response format: JSON
  • Parameters:
  • first (required): the name of the first place in the championship.
  • second (required): the name of the second place in the championship.
  • Example: api/championship/result?first=Dave&second=Armando
    Please wait to see the results after clicking
  • Example output: { status: "sucess" }

 

 

POST api/championship/top

Retrieves the top players of all championships. Returns the list of player names based on the count provided.

Resource information

  • HTTP Method: GET
  • Response format: JSON
  • Parameters:
  • count (optional): sets the count of records to be retrieved. If not provided, the default value is 10.
  • Example: api/championship/top?count=3
    Please wait to see the results after clicking
  • Example output: { players: ["Dave", "Armando", "Robert"] }

 

 

 

 

 

 

POST api/championship/new

Receives the championship data and computes it to identify the winner. The first and second place are stored into a database with their respectives scores. Returns the winner of the championship.

Resource information

  • HTTP Method: POST
  • Response format: JSON
  • Parameters:
  • data (required): The data of the championship following the bracketed array standard.
  • Example: api/championship/new?data=[["Armando","P"],["Robert","S"]]
    Please wait to see the results after clicking
  • Example output: { players: ["Dave", "Armando", "Robert"] }
